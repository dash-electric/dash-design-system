## Findings Summary

| Severity | Count |
|---|---|
| HIGH | 0 |
| MEDIUM | 1 |
| LOW | 2 |

### MEDIUM-1 ‚Äî PR body lacks editor identity (CR-2 governance gap)
- **File:** `src/pipeline/orchestrator.ts:469-494`
- **Issue:** `buildPRBody()` includes prompt id + timestamp (via PromptRecord) but does NOT include the OAuth `userEmail` (`src/auth/anthropic/client.ts:25`) of the team member who triggered the run.
- **Why it matters:** Pilot has 10+ team members. PR reviewer cannot attribute the prompt to a person from the PR alone ‚Äî must cross-reference server logs.
- **Fix:** Thread `userEmail` from `AnthropicCredentials` into `PromptRecord.editor` and into `buildPRBody` as `**Authored by:** ${userEmail}`. Small diff.

### LOW-1 ‚Äî `clarification/form-renderer.ts` duplicates token definitions
- **File:** `src/clarification/form-renderer.ts:13,86-91,123-170`
- **Issue:** Re-declares `#5e2aac`, `#fafafa`, `#1a1424`, `#b8336a` etc. instead of importing from a shared token module.
- **Why it matters:** If theme tokens drift in `daemon/templates/styles/dashboard.ts`, the clarification form will silently desync.
- **Fix:** Extract a shared `src/shared/tokens.ts` exporting hex constants; import from both `dashboard.ts` and `form-renderer.ts`.

### LOW-2 ‚Äî `preview/shell-renderer.ts` duplicates token definitions
- **File:** `src/preview/shell-renderer.ts:14-30`
- **Issue:** Same as LOW-1 ‚Äî duplicates Dash Purple + neutral tokens for the iframe fallback. Legitimate need (loads before consumer CSS) but duplication risk.
- **Fix:** Same shared `tokens.ts` module.
