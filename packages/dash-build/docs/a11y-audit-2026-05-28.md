# A11y Audit — Dash Control Tower HTML
Date: 2026-05-28
Scope: `packages/dash-build/docs/dash-control-tower.html` (light mode, single-file)
Standard: WCAG 2.1 AA
Status: 12 findings (3 HIGH / 5 MED / 4 LOW)

## TL;DR

HTML uses Dash DS semantic tokens correctly. Major a11y issues are **structural** (missing landmarks, no skip-link, no `lang` on dynamic content, button-as-button consistency), not visual. Color contrast passes AA on all critical surfaces. Keyboard nav works for tabs but **agent console controls miss focus indicators** in some states. No ARIA roles on the AI agent console — screen readers will hear "button button button" with no context.

---

## HIGH — fix before sharing externally

### H1. No skip-link to main content
- **Where**: top of `<body>`
- **Impact**: keyboard users tab through all 7 nav links before reaching the page
- **Fix**: add `<a href="#overview" class="skip-link">Skip to content</a>` styled with `position: absolute; left: -9999px; &:focus { left: 8px; top: 8px; }`

### H2. Missing semantic landmarks
- **Where**: `<div class="app-shell-topbar">`, `<div class="workspace">`, `<div class="footer">`
- **Impact**: screen readers can't jump between regions
- **Fix**: change to `<header class="app-shell-topbar" role="banner">`, `<main class="workspace" id="main-content">`, `<footer class="footer" role="contentinfo">`. Wrap nav in `<nav aria-label="Main">`.

### H3. AI agent console panes have no ARIA roles
- **Where**: `.agent-console-grid` with 3 `.agent-pane` divs
- **Impact**: screen reader users have no idea this is a multi-region live console. "streaming" pulse-dot is purely visual.
- **Fix**:
  - Add `role="region" aria-label="Thinking stream"` etc. to each pane
  - Add `aria-live="polite"` on `#ts` (thinking-stream container) so new blocks are announced
  - Add `aria-label="streaming"` + `aria-live="off"` (decorative) on `.avail-dot`
  - Wrap decision tree nodes in `<button>` not `<div>` (currently divs with `cursor:pointer`)

## MED — fix before next iteration

### M1. Tab buttons missing ARIA tab/tablist roles
- **Where**: `.tabs` + `.tab` + `.tab-content`
- **Fix**: add `role="tablist"`, `role="tab"`, `aria-selected="true|false"`, `aria-controls="tab-content-id"`, `role="tabpanel"`. Wire keyboard arrow-key navigation (currently only click works).

### M2. Decision tree nodes are clickable divs
- **Where**: `.dt-node`
- **Impact**: keyboard users cannot trigger rationale swap. No tab-stop, no Enter handler.
- **Fix**: change to `<button class="dt-node" type="button">`. Override default button styles. Or add `tabindex="0"` + Enter/Space key handler.

### M3. Mermaid diagrams have no text alternative
- **Where**: every `.mermaid-wrap`
- **Impact**: blind users see nothing
- **Fix**: add `<figure>` wrapper with `<figcaption class="sr-only">` summarizing the diagram in plain text. Or add `aria-label` describing the flow.

### M4. Focus indicator weak on lighter buttons
- **Where**: `.btn-lighter.btn-primary` (purple background, purple text)
- **Impact**: 4px ring (`--primary-alpha-10`) low contrast on white bg → invisible focus
- **Fix**: use `--primary-alpha-24` for ring opacity OR `outline: 2px solid var(--primary-base); outline-offset: 2px`

### M5. Streaming live region uses `<div>` with no `aria-live`
- **Where**: `#ts` thinking stream
- **Impact**: screen readers don't announce new content as agent thinks
- **Fix**: `<div id="ts" class="thinking-stream" role="log" aria-live="polite" aria-atomic="false" aria-relevant="additions">`

## LOW — nice-to-have

### L1. Typing cursor (`.cursor::after`) blinks 60 times/min
- **Impact**: WCAG 2.2.2 (Pause/Stop/Hide) — content blinking >5s with no off-switch is a violation
- **Fix**: respect `prefers-reduced-motion: reduce` → `animation: none`

### L2. Pulse animation on `.avail-dot`
- **Impact**: same as L1, motion-sensitive users
- **Fix**: media query `@media (prefers-reduced-motion: reduce) { .avail-dot::after { animation: none; } }`

### L3. Anchor offset using invisible `<span class="anchor">`
- **Impact**: hacky, conflicts with proper focus management
- **Fix**: use `scroll-margin-top: 64px` on `[id]` selectors instead — modern CSS, accessible

### L4. `<code>` inside table cells has no `tabindex` for long content
- **Impact**: minor — keyboard users can't scroll inline code horizontally on mobile if it overflows
- **Fix**: optional `tabindex="0"` on overflowing `<code>` or `<pre>`
