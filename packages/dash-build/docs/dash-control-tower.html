PRD — Dash Build v0.1 (beta)

Problem

Dash punya 10 PE tribes butuh internal tools (backoffice, dashboard, ops UI) cepat. Engineering bandwidth scarce. Designer-to-code handoff manual, lambat, drift dari DS.

Goal
  • ● Founder/PM bisa ngebuat draft halaman ops dalam <5 menit dari prompt.
  • ● Output 100% pakai komponen Dash DS — zero custom CSS drift.
  • ● Iframe preview real (bukan staging) sebelum PR.
  • ● PR auto-open di repo target dengan branch terisolasi.
Non-goals (v0.1)
  • ○ Bukan pengganti engineer untuk production-grade backend logic.
  • ○ Bukan WYSIWYG drag-n-drop editor (datang via Layer C).
  • ○ Bukan untuk external/customer-facing (internal Dash only).
Success metrics — target 30 hari setelah beta
MetricTargetCurrent

PRs opened/week10+tbd
DS adoption in generated code> 95%tbd
Founder/PM users active3+1 (Irfan)
Cost per successful PR< $0.50tbd
User stories
  • U1 Sebagai PM ops, gua mau type "halaman approve refund per outlet" dan dapet preview iframe yang real.
  • U2 Sebagai engineer, gua mau review PR yang dibuat AI dan pastikan ga ada hack untuk lolos type-check.
  • U3 Sebagai founder, gua mau lihat di Owner Dashboard berapa cost AI per minggu dan modul mana paling sering dibuat.
  • U4 Sebagai DS owner, gua mau tau component mana yang sering di-request tapi belum ada di registry.

PRD — Dash DS

Why exists

10 Dash PE/tribes (Dash Express, Halo-Dash, dll) butuh consistency. Tanpa DS = 10 versi button, 10 versi modal, 10 versi typography. DS = upstream truth.

Personas
  • D Designer — design di Figma pakai AlignUI Pro → submit ke Dash DS untuk port.
  • E Engineer tribe — install @dash-electric/*, import komponen, ga perlu re-implement.
  • B Dash Build AI agent — query registry untuk resolve user prompt ke komponen yang ada.

Stage-by-stage commands

Setup (one-time)

5 min

Install deps + auth OpenAI + auth GitHub.

cd /Users/irfanprimaputra.b/Work/dash/dash-ds
pnpm install
pnpm --filter @dash/build build
gh auth status   # verify GitHub auth ready

OpenAI: BYO key sk-... atau codex login --device-auth.

Boot daemon

2 min
cd packages/dash-build
node dist/bin.js
# tekan Enter → "Web UI" default
# browser auto-buka :7777/dashboard
  • ✓ Dashboard render section Connect OpenAI + GitHub + Pick Repo + Prompt
  • ✗ Kalau error: lsof -i :7777 cek port conflict

Connect auth

3 min
  • ① Paste OpenAI key sk-... di form → Submit → status hijau
  • ② Click "Install GitHub App" → install flow atau stub callback
  • ③ Dropdown repo → pilih backoffice atau portal-v2 (allow-listed)

Bootstrap sandbox

3-5 min

Click "Bootstrap". State flip: idle → clone_running → install_running → dev_starting → dev_live

  • 🟡 clone_running ~15-30s
  • 🟡 install_running ~60-180s (paling lama)
  • 🟡 dev_starting ~60-180s (next dev cold)
  • 🟢 dev_live · :3101 — iframe load tanpa HTTP 500 = validate W1.1 fix

Type prompt + generate

5-10 min

3 contoh prompt (pilih satu):

A) Halaman list mitra dengan filter status (active/suspended/
   pending), search by name, pagination. Total count di topbar.

B) Halaman approve refund per outlet. DataTable dengan kolom
   outlet, amount, status, requested_at. Bulk select + approve/
   reject action. Drawer kanan untuk detail per row.

C) Settings page dengan 3 tab: Account, Integrations, Team.

Skill chain jalan: dash-intake → dash-prd → dash-design → dash-trd → Skill v4 + Codex. Foundation score muncul di end.

Preview iframe

3 min
  • ✓ Route baru muncul (/mitra, /refunds, /settings)
  • ✓ Component pakai Dash DS (Button, DataTable, Badge, Drawer)
  • ✓ Bulk select → BulkActionBar muncul (kalau prompt B)
  • ! Data list = 4 mitra fixture, by design

Approve + PR

3 min

2 min
