Inti trunk-based untuk Dash Build

  • 1 trunk = backoffice main, always deployable, source-of-truth
  • Short-lived branches: dash-build/<userId>-<runId>, target lifetime < 24 jam, hard expire 7 hari
  • Ephemeral sandbox: clone destroyed atau recycled per run, bukan persistent workspace
  • Continuous integration: tiap publish auto-validate (cardinal rule + audit + tests) sebelum Owner inbox
  • Owner gate: AI triage 80%, manusia approve critical 20%
  • Auto-sweep: branch > 7 hari unmerged → auto-delete + notify creator
  • Preview-shim isolated: never lands di trunk; verified via cherry-pick exclude

Implementation checklist Phase D

Apa yang harus dibangun di daemon untuk activate flow di atas.

  • ✅ Bare clone reference setup (1x per project): ~/Work/dash-build-clones/<repo>/
  • 🔲 State machine module src/runs/sandbox-state.ts — implement state transitions (Clean → Cloned → ShimApplied → Idle → Generating → PreviewReady → Publishing → Sweep)
  • 🔲 Preview-shim patcher src/runs/preview-shim.ts — apply firebase noop + AuthContext bypass + apiService stub. Versioned per repo (backoffice v1.0, portal-v2 v1.0).
  • 🔲 Branch manager src/runs/branch-manager.ts — create dash-build/<userId>-<runId> from main HEAD, cherry-pick shim, commit generated.
  • 🔲 Publish extractor src/runs/publish.ts — git format-patch exclude shim, push via GitHub App, create PR with "DO NOT MERGE" shim marker.
  • 🔲 CI gate runner src/runs/ci-gate.ts — pre-push run audit:css + audit:tokens + test on the branch.
  • 🔲 Stale sweeper cron job — scan branches > 7 days, delete + notify.
  • 🔲 Conflict resolver src/runs/conflict.ts — auto-rebase, escalate kalau real conflict.
  • 🔲 State persistence di ~/.dash-build/state.json — track current sandbox state, recover on restart.
  • 🔲 Topbar mode badge — "Clone preview · main @abc1234 · shim v1.0 · sync 2h ago"
  • 🔲 Surface 3 wire — emit branch event ke owner dashboard saat publish.
